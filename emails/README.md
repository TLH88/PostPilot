# Emails

React Email templates that the app sends through Resend.

## Preview templates locally

```bash
npm run email
```

Opens a browser at `http://localhost:3001` with live-reloading previews of every `.tsx` file in this directory. Edit a template and the preview refreshes — no actual emails are sent.

## Send a real test email

POST to `/api/admin/email/test` (admin-only):

```bash
curl -X POST http://localhost:3000/api/admin/email/test \
  -H "Content-Type: application/json" \
  -d '{"to":"you@example.com","from":"hello"}'
```

`from` is one of `noreply`, `hello`, `news`.

## Add a new template

1. Create `emails/my-template.tsx` exporting a default React component.
2. Import it in the call site: `import { MyTemplate } from "@/emails/my-template"`.
3. Send via `sendEmail({ from, to, subject, react: MyTemplate(props) })`.

## Conventions

- One template = one purpose. Don't branch UI inside a template with `if (kind === ...)`.
- Use `@react-email/components` primitives (`Body`, `Container`, `Text`, etc.) — they're hand-tested for Gmail / Outlook / Apple Mail.
- `<Preview>` text shows in the inbox list before the user opens the email — write something useful.
- Plain-text fallback is auto-generated by Resend. Don't worry about it.
